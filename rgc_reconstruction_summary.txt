RGC Natural Movie Frame Reconstruction Summary

1. Task Definition

Our final task is:

- input: a short post-stimulus RGC population spike-response window
- target: the corresponding natural movie frame
- metrics: MSE, PSNR, SSIM, and GT-vs-reconstruction visualization
- split: scene/block split with embargo to reduce temporal leakage
- controls: shuffle response, mean response, zero response, mean frame, previous frame

The scientific question is:

Given the RGC population activity evoked by natural movies, how much information about the visual stimulus frame can be decoded back into an image?


2. From Reproduction to the Main Model

We did not jump directly into an optimized model. The work progressed in stages.

Stage A: Reproduction baseline

We first built a clean decoding pipeline that could run end to end:

- load movieBinnedSpiking.mat and AVI movie frames
- align frame t with the post-stimulus response window
- train a decoder to predict frame t from the aligned neural response
- evaluate with MSE / PSNR / SSIM / visualization

This stage was important because the earlier binaryCheckerboard -> whole-frame reconstruction route was scientifically mismatched to the natural movie reconstruction task.

Stage B: Better baseline

After the basic pipeline worked, we improved the baseline in a principled way:

- retained repeat-level samples instead of collapsing everything into PSTH too early
- used leakage-safe temporal split instead of random frame shuffle
- used temporal encoders + spatial decoders
- used RF-weighted losses and Gabor-style feature losses
- added control experiments to verify whether the model really used neural information

This gave us a stronger and more scientifically defensible baseline.

Stage C: Main model direction

Once the pipeline and baselines were stable, we observed a recurring problem:

- metrics could improve
- but the reconstructions still looked too smooth and blurry

That observation motivated the main model redesign.


3. Why We Started Thinking About Low and High Frequencies

From the visual results, the model often did this:

- recover the rough scene structure
- recover brightness layout and large shapes
- fail to reconstruct clear edges, textures, and fine details

This suggested that the learning difficulty was not uniform across frequency bands.

Low-frequency information:

- global brightness
- broad contours
- coarse layout

High-frequency information:

- edges
- local textures
- fine structures

These two parts behave differently:

- low-frequency structure is easier to stabilize
- high-frequency detail is much noisier and easier to overfit

So the intuition was:

If we force one model and one loss to handle both at once, the easiest solution is often to predict a safe, smooth average image. That improves MSE and sometimes SSIM, but does not produce sharper reconstructions.


4. Why Pure High/Low Frequency Splitting Was Not Enough

We then tried a more explicit frequency-split direction.

The idea was:

- one branch handles low-frequency structure
- another branch handles residual high-frequency information

This was useful conceptually, but pure splitting exposed a new issue.

What went right:

- training became more structured
- SSIM often became more stable
- the model more clearly separated coarse structure from residual detail

What went wrong:

- if high-frequency pressure was too strong, the model started amplifying noise-like residuals
- if low-frequency regularization was too weak, the model drifted toward unstable detail compensation
- if the frequency split was too rigid, the final reconstruction could still look oversmoothed or unnatural

In other words, pure frequency splitting was a useful idea, but by itself it was not the final answer.


5. Final Main Model Logic: Fusion Instead of Pure Splitting

The final direction that made the most sense was not "only low-frequency" or "only high-frequency", but fusion:

pred = coarse/base + spike-conditioned residual

This design means:

- the coarse/base branch is responsible for stable low-frequency structure
- the residual branch is responsible for spike-driven detail correction
- the residual branch should be encouraged, but not allowed to dominate blindly

This led to the detail-preserving residual decoder.

Its logic is:

- a base/coarse image handles the large-scale structure
- a residual branch adds neural-information-driven refinements
- training explicitly penalizes the model if it washes out edges and high-frequency detail

To support that, we added:

- low-frequency auxiliary regularization
- edge loss
- Laplacian loss
- high-frequency residual loss
- optional RF-aware residual gate

This is better than pure splitting because it does not force the final reconstruction to live inside a rigid low/high decomposition. Instead, it uses the decomposition as an inductive bias while still letting the final prediction be fused.


6. Why the Best Setting Was
   LOW_AUX_SIGMA = 2.0
   HIGHFREQ_LOSS_WEIGHT = 0.08

Among the tested detail-residual settings, the most balanced one was:

- LOW_AUX_SIGMA = 2.0
- HIGHFREQ_LOSS_WEIGHT = 0.08

Why this worked best:

1) LOW_AUX_SIGMA = 2.0

This gives the model a moderate low-frequency target.
It helps stabilize coarse structure without forcing the model to become too flat.

If low-frequency regularization is too weak:

- the model can become unstable
- the residual branch overcompensates

If it is too strong:

- the reconstruction becomes overly smooth

So sigma = 2.0 was a practical middle ground.

2) HIGHFREQ_LOSS_WEIGHT = 0.08

This gave enough pressure for detail preservation, but not so much that the model started hallucinating noisy high-frequency patterns.

When we pushed the high-frequency term higher, for example to 0.10 or 0.12:

- the paper metrics could still look fine
- SSIM could even become more consistently around 0.5
- but visually the reconstruction did not become clearly sharper
- in some cases it looked more forced or less natural

So the final interpretation is:

The model benefits from separating low- and high-frequency responsibilities, but the high-frequency branch must be kept under control. Otherwise it starts learning unstable residuals instead of meaningful detail.


7. What the Results Mean

The final results support several conclusions.

Conclusion A: the decoding pipeline is real

The normal model consistently outperformed several controls such as:

- shuffle response
- mean response
- zero response
- mean frame

This indicates that the model is not merely outputting a generic average image. It is using neural-response information.

Conclusion B: previous-frame baseline remains very strong

The previous-frame baseline is still much better than the learned decoder.
This means temporal continuity in natural movies is an extremely strong prior.

So we must be careful:

good reconstruction scores do not automatically mean the model recovered sharp, frame-specific neural detail. Some of the apparent success may come from broad scene continuity and low-frequency structure.

Conclusion C: current models mainly recover coarse scene structure

At this stage, the model is good at:

- large-scale layout
- brightness structure
- rough scene identity

But still weak at:

- sharp boundaries
- fine local textures
- crisp frame-specific details

So the most accurate scientific statement is:

The decoder recovers meaningful coarse visual information from RGC population responses, but high-fidelity fine-detail reconstruction remains limited.


8. Final Recommended Narrative for the Report

If this is written into the report, the recommended storyline is:

1) We first reproduced a leakage-safe natural-movie frame reconstruction pipeline from RGC population responses.

2) We then established stronger baselines using repeat-level neural responses, temporal encoders, spatial decoders, RF-weighted losses, and control experiments.

3) From the visual results, we observed that better MSE/PSNR/SSIM did not necessarily mean sharper reconstructions; many models improved metrics by producing smoother images.

4) This motivated a high/low-frequency interpretation of the task:

- low-frequency structure is easier to decode and stabilize
- high-frequency detail is noisier and easier to overfit

5) Pure frequency splitting was informative but not sufficient on its own, because overly aggressive high-frequency modeling produced unstable residuals and did not consistently improve perceptual clarity.

6) Therefore, we moved to a fused coarse-plus-residual model:

- base/coarse branch for structure
- spike-conditioned residual branch for detail
- auxiliary low-frequency supervision
- edge / Laplacian / high-frequency losses

7) The best balance came from:

- LOW_AUX_SIGMA = 2.0
- HIGHFREQ_LOSS_WEIGHT = 0.08

This setting did not fully solve the blur problem, but it gave the most reasonable balance between stable structure recovery and controlled detail preservation.


9. Short Final Takeaway

From reproduction to the final model, the central lesson was:

The challenge is not just to lower reconstruction error, but to stop the decoder from collapsing into a smooth average image.

High/low-frequency separation helped us understand that problem.
Pure frequency splitting was not enough by itself.
The most defensible final design was a fused model:

- stable coarse reconstruction
- spike-conditioned residual refinement
- explicit detail-aware losses

This gave the clearest path from replication to a scientifically motivated optimized decoder.
